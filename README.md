# World Cup Match Outcome Prediction

This project predicts international football match outcomes using historical
match data and machine learning.

## Problem

The goal is to predict one of three match outcomes:

- `0`: home team loses
- `1`: draw
- `2`: home team wins

## Dataset

The project uses two Kaggle datasets. Place them at:

```text
data/raw/results.csv                    # international football results
data/raw/fifa_ranking-2024-06-20.csv    # FIFA world rankings (1992вЂ“2024)
```

The pipeline keeps modern matches from 1990 onward, creates form-based features
from past matches only, and splits the data chronologically:

- Train: matches before 2022
- Test: matches from 2022 onward

## Models

Models are tuned by **macro F1** (not accuracy) using a chronological 80/20
inner split to avoid rewarding majority-class bias. All models use
`class_weight="balanced"` so that draws and home losses are not drowned out
by the more frequent home-win class.

The 29 features cover: rolling form per team per venue type (16), match
context (3), self-computed Elo ratings (2), FIFA rankings as of match date (4),
head-to-head history (2), and days since last match per team (2).

`xgboost` was not available in the local environment, so scikit-learn's
`GradientBoostingClassifier` was used as the boosting fallback.

## Main Results

With FIFA rankings, head-to-head record, and days-since-last-match added to
the feature set, accuracy now exceeds the naive baseline for all models.
**Macro F1 is the primary comparison metric.**

| Model | Accuracy | Macro F1 | CV Macro F1 (3-fold) |
| --- | ---: | ---: | ---: |
| Logistic Regression | 53.14% | **0.4983** | 0.530 В± 0.050 |
| Random Forest | **54.26%** | 0.4962 | 0.517 В± 0.056 |
| Gradient Boosting | 53.32% | 0.4955 | **0.533 В± 0.045** |
| Decision Tree | 51.55% | 0.4774 | 0.501 В± 0.055 |

Walk-forward CV (test years 2019, 2020, 2021): **LR, RF, and GB are
statistically indistinguishable** on the held-out test (all McNemar's p > 0.08).
Decision Tree is significantly weaker than RF and GB. Logistic Regression is
the recommended champion вЂ” equal performance with far greater interpretability.

## How to Run

Install dependencies:

```bash
pip install -r requirements.txt
```

Create the processed dataset (run from the repo root):

```bash
python src/data_pipeline.py
```

Train and evaluate all models:

```bash
python src/train_evaluate.py
```

Both scripts must be run from the repository root so that relative paths
(`data/`, `outputs/`) resolve correctly.
